Los 3 Pilares de la Sintaxis

Para escribir cualquier consulta en SQL, necesitas dominar tres componentes esenciales:

1. Keywords: Las acciones.
2. Data Types: La naturaleza de la información.
3. Operators: Las comparaciones.

2 / 19

Parte 1: Keywords

Las palabras clave son comandos reservados que le indican a la base de datos la operación a ejecutar.

3 / 19

Keywords de Consulta (DQL)

Las más usadas. Definen qué datos leer y cómo presentarlos.

SELECT Columnas  -- Qué quieres ver
FROM Tabla       -- De dónde
WHERE Condición   -- Cómo filtrar
ORDER BY Columna -- Cómo ordenar
4 / 19

Keywords de Manipulación (DML)

Usadas para crear, modificar o eliminar los registros de las tablas.

INSERT INTO Tabla ... VALUES ...;
UPDATE Tabla SET ... WHERE ...;
DELETE FROM Tabla WHERE ...;
5 / 19

Keywords de Definición (DDL)

Definen la estructura de la base de datos: las tablas y sus columnas.

CREATE TABLE Nombre (...);
ALTER TABLE Nombre ...;
DROP TABLE Nombre;
6 / 19

Parte 2: Tipos de Datos

Definir correctamente el tipo de dato ahorra espacio y evita errores. Vamos a verlos en detalle.

7 / 19

Números Enteros

Se usan para contar cosas (stock, edad, IDs). No tienen decimales.

-- INT (o INTEGER): El estándar.
-- SMALLINT: Para números pequeños.
-- BIGINT: Para cifras muy grandes.

Edad INT, Cantidad SMALLINT
8 / 19

Números Decimales

Cruciales para precios, pesos o medidas científicas.

-- DECIMAL(p, s): Exacto. Ideal para DINERO.
-- FLOAT / REAL: Aproximado. Para ciencia.

Precio DECIMAL(10, 2)
9 / 19

Texto Corto (Strings)

Para nombres, códigos, emails, etc.

-- VARCHAR(n): Longitud Variable (común).
-- CHAR(n): Longitud Fija (códigos).

Nombre VARCHAR(100), CodigoPais CHAR(2)
10 / 19

Texto Largo

Cuando `VARCHAR` se queda corto (descripciones, comentarios).

-- TEXT: Para párrafos largos.
-- LONGTEXT / CLOB: Para textos enormes.

Descripcion TEXT, ContenidoBlog LONGTEXT
11 / 19

Fechas y Tiempo

Usa el formato 'YYYY-MM-DD'.

-- DATE: Solo fecha (2023-12-31).
-- TIME: Solo hora (23:59:59).
-- DATETIME: Ambos (fecha y hora).

FechaNacimiento DATE, UltimoAcceso DATETIME
12 / 19

Otros Tipos Comunes

Tipos especializados para necesidades concretas.

-- BOOLEAN: Verdadero o Falso (TRUE/FALSE).
-- BLOB: Para guardar imágenes o archivos.
-- JSON: Para guardar objetos JSON.

EsActivo BOOLEAN, FotoPerfil BLOB

Parte 3: Operators

Los operadores son símbolos especiales que se usan para realizar comparaciones, cálculos y operaciones lógicas.

14 / 19

Operadores Aritméticos

Realizan cálculos matemáticos básicos.

+ (Suma), - (Resta), * (Multiplicación), / (División)

-- Calcular el precio con IVA (ej. 21%)
SELECT Nombre, Precio * 1.21 AS PrecioFinal FROM Productos;
15 / 19

Operadores de Comparación

Comparan dos valores y devuelven verdadero o falso.

=, <>, !=, >, <, >=, <=

-- Empleados con salario mayor o igual a 50000
SELECT * FROM Empleados WHERE Salario >= 50000;
16 / 19

Operadores Lógicos

Se usan para combinar múltiples condiciones.

AND -- Todas las condiciones deben cumplirse.
OR  -- Al menos una condición debe cumplirse.
NOT -- Invierte el resultado.

-- Clientes de España O con más de 5 compras
SELECT * FROM Clientes
WHERE Pais = 'España' OR NumeroCompras > 5;
17 / 19

Operadores Especiales

Proporcionan formas más potentes de filtrar.

IN (list) -- Valor en una lista.
BETWEEN x AND y -- Valor en un rango.
LIKE 'patron' -- Coincidencia de texto.
IS NULL -- El valor está vacío.

SELECT * FROM Productos WHERE Categoria IN ('Libros', 'Música');
18 / 19

Poniéndolo Todo Junto

Una consulta real combina todos estos elementos.

-- Keyword + Columna + Tabla + Keyword + Comparación + Lógico
SELECT Nombre, Salario 
FROM Empleados
WHERE Departamento = 'Ventas' 
  AND Salario BETWEEN 40000 AND 60000;
